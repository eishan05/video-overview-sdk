- [ ] 4. Script Generator (`src/video_overview/script/generator.py`): Create `ScriptGenerator` class. Method `generate(content_bundle, topic, mode, llm_backend, max_segments=20)` that: builds a detailed prompt asking the LLM to create a NotebookLM-style conversational script (for "conversation" mode) or narration script (for "narration" mode), includes the content bundle in the prompt, calls `claude` CLI via `subprocess.run` with `claude -p "prompt" --output-format json` (or `codex exec "prompt"` for codex backend), parses the JSON response into a `Script` model, validates all segments have required fields. The prompt should instruct the LLM to output a JSON object matching the `Script` schema with engaging educational dialogue between Host and Expert, where each segment includes a `visual_prompt` describing an informative diagram/illustration to accompany the narration. For narration mode, use a single speaker. Handle subprocess errors, timeout (5 minutes), and JSON parse failures with clear error messages. Write tests in `tests/test_script_generator.py` that mock subprocess calls and verify prompt construction and response parsing.

- [ ] 5. Audio Generator (`src/video_overview/audio/generator.py`): Create `AudioGenerator` class. Method `generate(script, host_voice, expert_voice, narrator_voice, cache_dir)` that: takes a Script model, chunks segments into batches of ~10-15 segments per API call, for each batch builds the TTS prompt with speaker labels, calls Gemini TTS API using `google.genai.Client().models.generate_content()` with model `gemini-2.5-flash-preview-tts`, `response_modalities=["AUDIO"]`, and appropriate `SpeechConfig` (use `MultiSpeakerVoiceConfig` for conversation mode, single `VoiceConfig` for narration mode), extracts audio bytes from response `inline_data`, saves each chunk as WAV file, concatenates all WAV chunks using ffmpeg (`ffmpeg -f concat -safe 0 -i list.txt -c copy output.wav`), returns the final audio file path and timing metadata (estimated duration per segment based on text length). Handle PCM-to-WAV conversion if mime_type is `audio/pcm`. Add retry logic with exponential backoff for API errors. Write tests in `tests/test_audio_generator.py` that mock the genai client and verify chunking, API call construction, and audio file handling.

- [ ] 6. Visual Generator (`src/video_overview/visuals/generator.py`): Create `VisualGenerator` class. Method `generate(script, cache_dir)` that: takes a Script model, for each segment's `visual_prompt` calls Gemini image generation using `google.genai.Client().models.generate_content()` with an image-capable model (try `gemini-2.0-flash-exp`, fallback list in config), passes prompt requesting a 16:9 informative diagram/illustration, extracts image from response `inline_data` and saves as PNG, runs up to 3 generations concurrently using `asyncio.gather` with semaphore, caches images by hash of visual_prompt to avoid regeneration, returns list of image file paths ordered by segment. If image generation fails for a segment, create a fallback image: solid dark background (#1a1a2e) with the segment text rendered in white using ffmpeg (`ffmpeg -f lavfi -i color=c=#1a1a2e:s=1920x1080:d=1 -vf "drawtext=text='...':fontsize=36:fontcolor=white:x=(w-text_w)/2:y=(h-text_h)/2" fallback.png`). Write tests in `tests/test_visual_generator.py` that mock the genai client and verify concurrent generation, caching, and fallback behavior.

- [ ] 7. Video Assembler (`src/video_overview/video/assembler.py`): Create `VideoAssembler` class. Method `assemble(audio_path, image_paths, segment_durations, output_path, format)` that: for "audio" format, converts WAV to MP3 using ffmpeg and copies to output_path; for "video" format, creates an ffmpeg filter complex that: assigns each image a duration matching its segment's audio duration, applies a subtle Ken Burns effect (slow 5% zoom over duration using `zoompan` filter), adds 0.5s crossfade transitions between images using `xfade` filter, overlays the audio track, outputs H.264+AAC MP4 at 1920x1080 30fps. Method `_estimate_segment_durations(segments, total_audio_duration)` that estimates each segment's duration proportional to its text length relative to total text. Validate ffmpeg is available at init. Clean up temp files. Write tests in `tests/test_video_assembler.py` that mock subprocess/ffmpeg calls and verify filter construction and command building.
